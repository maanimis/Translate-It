# راهنمای بومی‌سازی

برای بومی‌سازی **Translate It!** به زبان خود، لطفاً مراحل ذکر شده در این سند را دنبال کنید.

---

## نحوه‌ی مشارکت در ترجمه

در اینجا ساده‌ترین روش برای انجام این کار آورده شده است:

### ۱. کپی کردن پروژه (Clone)
ابتدا پروژه را در سیستم خود کپی کنید:
```bash
git clone https://github.com/iSegar0/Translate-It.git
```

### ۲. پیدا کردن زبان مورد نظر
تمام فایل‌های ترجمه در پوشه‌ی `_locales/` قرار دارند.
- اگر زبان شما از قبل وجود دارد (مثلاً `_locales/fa/` برای فارسی)، می‌توانید با بهبود ترجمه‌های موجود کمک کنید.
- اگر وجود ندارد، ما می‌توانیم در راه‌اندازی آن به شما کمک کنیم!

### ۳. ترجمه عبارات
ترجمه‌ها در فایلی به نام `messages.json` ذخیره می‌شوند. ساختار آن به این صورت است:

```json
"settings_title": {
  "message": "Settings"
}
```

**وظیفه‌ی شما:**
- متن داخل `"message"` (مثلاً `"Settings"`) را به زبان خود تغییر دهید.
- **مهم:** کلید سمت چپ (مثلاً `"settings_title"`) را به هیچ وجه تغییر ندهید.

### ۴. متغیرهای خاص
برخی از پیام‌ها دارای متغیرهایی مانند `{appName}` یا `{version}` هستند.
- لطفاً این موارد را دقیقاً همان‌طور که هستند (شامل آکولادها) نگه دارید تا افزونه بتواند اطلاعات صحیح را به طور خودکار جایگزین کند.

---

## برای کاربران حرفه‌ای (اسکریپت‌ها)

اگر با ترمینال راحت هستید، ما ابزارهایی برای آسان‌تر کردن کارها داریم:

۱. **همگام‌سازی کلیدها:** دستور `pnpm run i18n:sync` را اجرا کنید تا ببینید آیا کلمات انگلیسی جدیدی وجود دارد که نیاز به ترجمه داشته باشد.
۲. **افزودن خودکار کلیدهای ناقص:** دستور `pnpm run i18n:sync:fix` را اجرا کنید تا کلیدهای ناقص از فایل انگلیسی به فایل زبان شما اضافه شوند.
۳. **پاک‌سازی:** پس از اتمام ترجمه، دستور `pnpm run i18n:purge [کد-زبان-شما]` را اجرا کنید تا یادداشت‌های "UNTRANSLATED" حذف شوند.

## ارسال ترجمه‌ی شما

زمانی که از ترجمه‌های خود راضی بودید:
۱. یک **Pull Request** در GitHub ارسال کنید.
۲. ما آن را بررسی کرده و در به‌روزرسانی بعدی قرار می‌دهیم!

*اگر مراحل فنی (مانند GitHub یا PR) برای شما دشوار است، راحت باشید و یک **Issue** باز کنید و فایل `messages.json` ترجمه شده‌ی خود را در آنجا پیوست (attach) کنید. ما بقیه‌ی کارها را انجام می‌دهیم!*
